# restaurant_insight_tool
Restaurant Insight-to-Action Engine
A Streamlit-based analytics tool that turns unstructured restaurant reviews into clear business themes and owner-ready action plans.
Overview
The workflow takes restaurant review text, sanitizes obvious PII, converts the text into TF-IDF features, clusters reviews into themes using KMeans, and generates prescriptive recommendations that a restaurant owner or marketing manager can act on.
The tool is designed to be:
Explainable rather than black-box
Offline/demo friendly with no paid API dependency
Practical for marketing and operations decisions
What the tool does
The app supports three main stages:
Upload Reviews
Upload a CSV file of restaurant reviews
Validate required columns
Preview the data and optional health checks
Analyze & Generate Strategy
Sanitize review text to reduce obvious personal information such as emails, phone numbers, URLs, and long numeric IDs
Convert reviews into TF-IDF vectors
Group similar reviews into themes using KMeans clustering
Generate a strategy table with:
theme names
review counts
average ratings
evidence quotes
fix/promote guidance
week-by-week content recommendations
Dashboard
Quick stats
Theme distribution chart
Average rating by theme
Ratings distribution
Reviews per restaurant

Methods used
1. Text sanitization
The pipeline masks obvious PII patterns such as:
email addresses
phone numbers
URLs
long numeric IDs
2. Text representation
The cleaned review text is transformed using TF-IDF so the model can measure which words and phrases are most important across reviews.
3. Theme discovery
The project uses KMeans clustering to group similar reviews into themes such as service, speed, cleanliness, value, or food quality.
4. Strategy generation
The tool translates theme insights into business-friendly recommendations such as:
what to fix first
what to promote
what kind of content to post
a 2, 4, or 8 week promotional plan
Why this is useful
Restaurant reviews contain valuable business signals, but owners usually do not have time to read hundreds of comments manually. This tool helps convert free-text reviews into structured and actionable decisions for:
operations improvement
service recovery
marketing campaigns
local search competitiveness
Explainability / sanity checks
The project includes simple checks so the output stays interpretable:
top keywords should match the example evidence quotes
low-rated themes are treated as issues to fix first
high-rated themes are treated as strengths to promote
repeated clustering on subsets can be used to check whether the themes are reasonably stable
Installation
1. Create and activate a virtual environment
Windows
```bash
python -m venv .venv
.venv\Scripts\activate
```
Mac/Linux
```bash
python3 -m venv .venv
source .venv/bin/activate
```
2. Install dependencies
```bash
pip install -r requirements.txt
```
Run the app
Option 1: Streamlit command
```bash
streamlit run new_app.py
```
Option 2: Windows batch file
Double-click:
```bash
run_rest_tool.bat
```

Limitations
KMeans themes depend on the chosen number of clusters
Short or noisy reviews may reduce theme quality
The strategy is rule-based rather than generated by a live LLM
Sanitization reduces only obvious PII patterns, not all privacy risk
Results are best when analyzing a focused set of reviews for one restaurant or a small group of restaurants
